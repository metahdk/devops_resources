### Example Pipeline code 


pipeline {
    agent any
    stages {
        stage ('SCM for Java Code') {
            steps{
                git changelog: false, url: 'https://github.com/pavansw/simpleMavenJunit.git'
                echo 'Polling to Developers SCM'
            }
        }
        stage ('Maven Compile') {
            steps{
                sh '''mvn clean
mvn compile'''
            }
        }
        stage ('Testing Code') {
            steps {
                sh '''mvn test'''
                echo 'Testing complete with Junit on maven'


            }
        }
        stage ('Package') {
            steps {
            sh 'mvn package'   
            }
        }
        stage ('Publish Test results'){
            steps {
                
    
            junit 'target/surefire-reports/*.xml'
            }
        }
        stage ('Execute Jar'){
            steps{
                sh 'java -jar target/*.jar'
            }
        }
    }
}

#### TRYING AWS Cloud services for Container services 


Phase 1: IAM User Setup (In AWS Console)
   1. Access Account: Log in with the root or admin credentials provided.
   2. Create User: Go to IAM > Users > Create user.
   * Name: container-admin
   * Access: Check "Provide user access to the AWS Management Console".
   3. Assign Policies: Select Attach policies directly and add these three:
   * AdministratorAccess (As per your request to avoid nesting issues).
   * AmazonEC2ContainerRegistryFullAccess
   * AmazonECS_FullAccess
Ref :   
   4. Security Credentials: Once created, click on the user -> Security credentials tab -> Create access key -> Select Command Line Interface (CLI). Save the Access Key and Secret Key.
________________


Phase 2: Workstation Setup (Ubuntu Server)
Install AWS CLI:
Bash
sudo apt update && sudo apt install awscli -y
   1. Configure CLI:
Bash
aws configure
# Use the Access Key and Secret Key from Phase 1
# Default region: ap-southeast-2
# Default output: json
   2. ________________


Phase 3: ECR & Docker (Building the Image)
Create Repository:
Bash
aws ecr create-repository --repository-name my-web-app --region ap-southeast-2
   1. Login to ECR:
Bash
aws ecr get-login-password --region ap-southeast-2 | docker login --username AWS --password-stdin <YOUR_ACCOUNT_ID>.dkr.ecr.ap-southeast-2.amazonaws.com
   2. Build Apache Image: Create a Dockerfile:
vim Dockerfile


FROM ubuntu:latest
RUN apt-get update && apt-get install -y apache2
ENV APACHE_RUN_USER www-data
ENV APACHE_RUN_GROUP www-data
ENV APACHE_LOG_DIR /var/log/apache2
EXPOSE 80
CMD ["/usr/sbin/apache2ctl", "-D", "FOREGROUND"]


Build it:
Bash

docker build -t my-web-app .
   3. Tag & Push:
Bash

docker tag my-web-app:latest <YOUR_ACCOUNT_ID>.dkr.ecr.ap-southeast-2.amazonaws.com/my-web-app:latest
docker push <YOUR_ACCOUNT_ID>.dkr.ecr.ap-southeast-2.amazonaws.com/my-web-app:latest
   4. ________________


Phase 4: ECS Deployment (AWS Console)
   1. Login: Log in to the Console as container-admin.
   2. Create Cluster: Go to ECS > Clusters > Create. Use the Fargate (Serverless) infrastructure. Name it demo-cluster.
   3. Task Definition: * Name: web-task. Infrastructure: Fargate.
   * Container details: Name it apache-container.
   * Image URI: Paste your ECR URI from Phase 3.
   * Port Mapping: 80 (HTTP).
   4. Create Service: Inside demo-cluster, click Create under the Services tab.
   * Select your web-task.
   * Networking: Choose a VPC and Public Subnets.
   * Public IP: Ensure it is toggled to Enabled.
   * Security Group: Add an Inbound Rule for HTTP (Port 80) from 0.0.0.0/0.
________________


Phase 5: Verification & Cleanup
   1. Access: Go to Cluster > Tasks > Click your running Task. Copy the Public IP and paste it into your browser.
   2. Cleanup (Order matters):
   * ECS Service: Update service -> Set "Desired Tasks" to 0. Then Delete the service.
   * ECS Cluster: Delete the cluster.
   * ECR: Delete the my-web-app repository (this also deletes the images).
   3. Bash
aws ecr delete-repository --repository-name my-web-app --force
